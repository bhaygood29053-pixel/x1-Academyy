# Curriculum

## Track A — X1 Fundamentals (MVP)
1. What is X1? (SVM basics)
2. Accounts & programs (state model)
3. Transactions, compute, fees (why CU matters)
4. PDAs & deterministic state
5. RPC vs Indexers (why your app needs both)

## Track B — Xen on X1 (Post‑MVP)
1. Xen basics
2. Xen mechanics on X1 (where applicable)
3. Common misconceptions
4. Ecosystem map + safety

## Lesson format (standard)
Each lesson should include:
- A short story / explanation (3–7 minutes)
- 1–2 interactive checkpoints
- A quiz (5–10 questions)
- Optional puzzle (1–3 minutes)

## Difficulty ramp
- Beginner: vocabulary + concepts
- Intermediate: how to reason about accounts/transactions
- Advanced: tradeoffs, design patterns, failure modes
